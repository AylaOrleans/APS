# 1 Introdução

O levantamento de requisitos é uma etapa fundamental no desenvolvimento de sistemas, pois permite identificar as necessidades dos usuários e compreender os processos existentes dentro de uma organização.

Este documento apresenta o plano de levantamento de requisitos para o desenvolvimento de um sistema de gerenciamento de biblioteca. O objetivo é identificar os principais processos envolvidos, os problemas existentes no sistema atual e as funcionalidades desejadas para um novo sistema.

---

# 2 Definição de Stakeholders

Os stakeholders são as pessoas ou grupos que possuem interesse direto ou indireto no sistema.

Principais stakeholders identificados:

- Bibliotecários  
- Alunos  
- Professores  
- Coordenação da instituição  
- Setor de Tecnologia da Informação (TI)  
- Administração da biblioteca  

---

# 3 Técnicas de Levantamento de Requisitos

Para o levantamento de requisitos do sistema da biblioteca serão utilizadas algumas técnicas.

## 3.1 Entrevista

A entrevista será utilizada para conversar diretamente com os bibliotecários e compreender como funciona o trabalho atualmente dentro da biblioteca.

Essa técnica permite coletar informações detalhadas sobre os processos, identificar problemas existentes e entender as necessidades dos usuários.

**Observação:**  
A entrevista será aplicada caso seja possível realizar a visita presencial à biblioteca.

---

## 3.2 Questionário

O questionário será utilizado como uma técnica complementar para coletar informações de forma rápida e estruturada.

Essa técnica possui as seguintes vantagens:

- Aplicação rápida  
- Fácil organização das respostas  
- Facilidade na análise dos dados coletados  

---

## 3.3 Análise de Documentos

A análise de documentos consiste em examinar documentos já existentes na biblioteca para compreender regras, processos e informações importantes.

Essa técnica permite comparar informações e identificar requisitos diretamente a partir da documentação utilizada atualmente.

**Observação:**  
Apesar de não ser a primeira técnica recomendada, devido à possibilidade de não haver entrevista por motivos externos e conforme orientação do professor, esta técnica receberá maior atenção no processo de levantamento.

# 6 Questionário

O questionário será utilizado para coletar informações adicionais sobre o funcionamento da biblioteca e identificar necessidades para o novo sistema.

---

## 6.1 Informações Gerais

- Qual é o seu cargo na biblioteca?  
- Há quanto tempo você trabalha na biblioteca?  
- Quantas pessoas trabalham atualmente na biblioteca?  
- Quais são as principais atividades realizadas diariamente?  
- Existe algum sistema informatizado atualmente? Qual?

---

## 6.2 Cadastro de Livros

- Como os livros são cadastrados atualmente?  
- Quais informações são registradas no cadastro de um livro?  
- Quem é responsável por cadastrar novos livros?  
- Existe controle de quantidade de exemplares de cada livro?  
- Como é feito o controle quando um livro é perdido ou danificado?

---

## 6.3 Pesquisa de Livros

- Como os alunos ou professores pesquisam livros atualmente?  
- Existe um catálogo online ou a pesquisa é feita presencialmente?  
- Quais informações os usuários precisam ver ao pesquisar um livro?  
- O sistema mostra se o livro está disponível ou emprestado?  
- Existem dificuldades na busca de livros atualmente?

---

## 6.4 Empréstimo de Livros

- Como é realizado o empréstimo de um livro?  
- Quem pode pegar livros emprestados?  
- Quantos livros cada pessoa pode pegar emprestado?  
- Qual é o prazo para devolução?  
- O sistema registra automaticamente o empréstimo ou é manual?

---

## 6.5 Reservas de Livros

- Existe possibilidade de reservar livros que estão emprestados?  
- Como a reserva é feita atualmente?  
- Quanto tempo um livro reservado fica disponível para retirada?  
- O usuário recebe alguma notificação quando o livro fica disponível?

---

## 6.6 Devolução de Livros

- Como funciona o processo de devolução?  
- O sistema registra automaticamente a devolução?  
- O que acontece quando o usuário devolve o livro com atraso?  
- Existe multa por atraso?  
- Como essa multa é calculada?

---

## 6.7 Controle de Usuários

- Como os usuários são cadastrados no sistema?  
- Quais informações são armazenadas sobre os usuários?  
- O sistema diferencia alunos, professores e funcionários?  
- Existe limite diferente de empréstimo para cada tipo de usuário?

---

## 6.8 Problemas do Sistema Atual

- Quais são os principais problemas no sistema atual da biblioteca?  
- Existe algum processo que ainda é feito manualmente?  
- Já ocorreram erros no controle de empréstimos ou devoluções?  
- O sistema atual é fácil de usar para os funcionários?

---

## 6.9 Funcionalidades Desejadas

- Quais funcionalidades um novo sistema deveria ter?  
- Seria importante que os alunos pudessem pesquisar livros pela internet?  
- Seria útil receber notificações de devolução por e-mail ou aplicativo?  
- O sistema deveria gerar relatórios? Quais?

---

## 6.10 Considerações Finais

- Existe alguma funcionalidade importante que não foi mencionada?  
- O que você mudaria no sistema atual da biblioteca?  
- Há alguma sugestão para melhorar o atendimento aos usuários?

---

# 7 Considerações Finais

Este plano de levantamento de requisitos apresenta as técnicas e instrumentos que serão utilizados para coletar informações sobre o funcionamento da biblioteca. A combinação de entrevistas, questionários e análise de documentos permitirá compreender melhor os processos atuais e identificar requisitos importantes para o desenvolvimento de um novo sistema de gerenciamento de biblioteca.
